Tests cover acquire/release/double-acquire/release-without-acquire/contention-handoff/dispose/empty-name; 9/9 passing on real Win32 mutexes.
  Spec ref: `spec.md > §5.1 IMutexHolder`, `§4 Architecture > External boundaries`

Use CsWin32 specifically because we need explicit handle ownership semantics — `System.Threading.Mutex` does not give us the same control over abandoned-mutex behavior. Public surface: `bool IsHeld`, `bool Acquire()`, `void Release()`, `event EventHandler MutexLost`. `MutexLost` polled on a 5-second `System.Timers.Timer` checking the handle is still valid via `WaitForSingleObject` with timeout 0.
  Acceptance: `Acquire()` returns true and `IsHeld == true`. A second process attempting `Acquire()` against the same name returns false. `Release()` frees the handle (verified by Process Explorer or a sibling test process succeeding on `Acquire()` after). `MutexLost` fires when the handle is forcibly closed externally.
