## 3. Next build — "stay active" (two parts)

**The problem:** accounts idle-time-out ~20 min; a batch launched together → their auto-reconnects fire together → synchronized reconnect wave trips the trust gate. RoRoRo has NO hook into a running client (reconnect is client↔Roblox), so it can't harden the reconnect — the fix is keeping them active, which is input automation, which is a **plugin**.

### Governance (settled this session)
The "no macros" wall is **core-only**. The plugin system already hosts input automation via consent-gated capabilities — `rororo-ur-task` is the precedent. So keep-active is a plugin, split:

### Part A — Core window-activity awareness (THIS repo)
- **Goal:** track per-account **last-foreground / time-since-last-activity**; expose to plugins via a host capability/event.
- **Primitive to copy:** `rororo-ur-task/src/PluginHost/ForegroundWatcher.cs` — `GetForegroundWindow()` → pid → account (stateless, on-demand). Core already has the window handles (`RobloxWindowDecorator`, `Tray/RunningRobloxScanner`) + the `account-launched/exited` events.
- **Add:** a last-active-at timestamp per managed account (poll foreground or hook), plus a new host capability/event (e.g. `host.events.window-activity` or a queryable "last active at") in `src/ROROROblox.PluginContract/` + `src/ROROROblox.App/Plugins/PluginHostService.cs`.
- Pure observation → **wall-clean**. Also feeds detect/surface for the Limited + reconnect story.

### Part B — New simple keep-active plugin (NEW repo)
- **Template:** clone `rororo-ur-task` structure. Namespace pattern `Labs626.*`; separate EXE; named pipe `\\.\pipe\rororo-plugin-host`; consent-gated.
- **Minimum acting:** `Win32Focus` (focus the window) + `SendInput` **VK_SPACE** (jump), per-account, only when idle past a threshold (driven by Part A's awareness).
- **Capabilities:** `system.synthesize-keyboard-input` (disclosure-only, consent-gated) + `host.events.account-launched`/`account-exited` + the new awareness capability. **NO** recording, sequences, or OCR.
- **UI:** simple from step one — per-account on/off + idle threshold. That's the whole point vs ur-task's full suite.
- Power users stay on `ur-task` and record a focus+space macro; this is the no-frills sibling.
